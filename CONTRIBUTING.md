# Contributing Guidelines

Thank you for your interest in contributing to the **Electric Vehicle Tableau Dashboard** project! 🚗⚡  
We welcome contributions of all kinds — from bug fixes and new features to documentation and visualization improvements.

---

## 🛠 How to Contribute

1. **Fork the repository**  
   Create your own copy of the repo under your GitHub account.

2. **Clone your fork**  
   ```bash
   git clone https://github.com/your-username/ev-tableau-dashboard.git
3. **Create a branch  
Use a descriptive branch name:
git checkout -b feature/add-new-kpi

4. **Make your changes
Follow coding and visualization standards.
Add or update Tableau dashboards, datasets, or documentation.
Ensure your changes are well‑documented.

5. **Commit your changes  
Write clear, concise commit messages:
git commit -m "Add KPI for EV adoption growth by year"

6. **Push your branch
git push origin feature/add-new-kpi

7. **Open a Pull Request (PR)
Provide a clear description of your changes.
Reference related issues (e.g., Fixes #12).
Ensure all checks pass before requesting review.

## ✅ Contribution Areas
Data Cleaning: Improve preprocessing scripts or dataset preparation.
Dashboard Enhancements: Add new charts, KPIs, or filters.
Documentation: Improve README, tutorials, or usage guides.
Bug Fixes: Resolve issues in dataset handling or visualization logic.

## 📖 Reporting Issues
Use the Issues tab to report bugs or request features.
Provide steps to reproduce bugs.
Include screenshots or logs when possible.

## 🤝 Community Standards
By contributing, you agree to follow our Code of Conduct.
Respectful communication and collaboration are expected at all times.

## 📌 Additional Notes
Major changes should be discussed in an issue before starting work.
Contributions are reviewed by maintainers before merging.
We may request changes or improvements before approval.
